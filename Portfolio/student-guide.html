Tips for Success

DO:

  • Take photos while you work
  • Save your code frequently
  • Test one thing at a time
  • Use Serial Monitor to debug
  • Ask for help when stuck

DON'T:

  • Change multiple things at once
  • Skip the weekly reflections
  • Copy code without understanding it
  • Leave wires loose
  • Wait until Friday to document

Code Snippets You'll Learn

Ultrasonic Distance

long distance = pulseIn(echoPin, HIGH) * 0.034 / 2;

Servo Control

myServo.write(90); // Move to 90 degrees

Claw Functions

void openClaw() { clawServo.write(OPEN_POS); }
void closeClaw() { clawServo.write(CLOSED_POS); }

Weekly Reflection Examples

Good Example:
"I wired the ultrasonic sensor to pins 2 and 3, then wrote code to measure distance. My first readings were wrong because I had TRIG and ECHO swapped. After fixing that, I tested at 10cm, 20cm, 30cm and my accuracy was within 1cm."
Avoid:
"Worked on the robot. Did some wiring. It kind of works now."

Getting Help

When you're stuck:

  1. Check your wiring - Most problems are loose connections
  2. Use Serial.println() - Print values to see what's happening
  3. Read the error message - Arduino IDE tells you what's wrong
  4. Check the Resources page - Sample code and troubleshooting guides
  5. Ask a classmate - They might have solved the same problem
  6. Ask the teacher - Bring your code and describe what you tried

Photo Evidence Ideas

Practical Day Tips

Before your practical:
  • Charge your batteries fully
  • Test everything the day before
  • Tighten all screws and connections
  • Have backup code saved
  • Know how to quickly adjust settings
