Practical Test Rubric

Unit 5 — Claw Project | Deliverable 9 Component

Assessment: Live Hardware Demo
Day: Day 10 (Final)
Total Points: 50 pts

Overview

Students demonstrate their working claw system on real hardware in a live, in-class evaluation. The grader observes and scores each criterion in real time. No re-dos are permitted after the session ends. Code must run on the student's own microcontroller without instructor intervention. The 50-point practical score is one component of the 75-point Deliverable 9 grade; the remaining 25 points come from the Final Design Brief submission.

1 — Startup & Hardware (8 pts)
Criterion Performance Description Pts
Physical Assembly
4 Claw is fully assembled; no loose wires; all components mounted securely.
2–3 Minor issues (one loose wire, component slightly unsecured) but functional.
1 Functional but sloppy build; multiple loose connections.
0 Non-functional or not assembled.
 4
Code Uploads & Runs
4 Student uploads and runs code independently; claw responds within 30 seconds of power-on.
2–3 Runs with one minor prompt from instructor (e.g., reminder to select COM port).
1 Requires significant instructor assistance to get running.
0 Code does not run.

Section Total/ 8
2 — Core Functionality (18 pts)
Criterion Performance Description Pts
Servo Control (Open / Close)
6 Claw opens to a fully open position and closes smoothly; motion is controlled (not jerky or slamming).
3–5 Claw opens and closes but motion is inconsistent or range is limited.
1–2 Partial movement only (opens or closes but not both reliably).
0 Servo does not respond.
 6
Potentiometer Feedback to Serial
6 Serial Monitor displays live ADC readings that visibly change as claw opens/closes; values make sense (e.g., 0–1023 range or mapped equivalent).
3–5 Readings appear but are noisy, stuck, or update only intermittently.
1–2 Serial Monitor prints something but values do not reflect claw position.
0 No Serial Monitor output.
 6
Contact Detection
6 Claw autonomously stops closing when it contacts an object (does not crush); demonstrated on at least 2 different objects.
3–5 Contact detection works on 1 object reliably, or works on 2 objects inconsistently.
1–2 Some evidence of detection logic but claw overshoots or misses contact most of the time.
0 No contact detection; claw stalls or requires manual stop.
 6
Section Total/ 18
3 — Object Classification & LED Feedback (12 pts)
Criterion Performance Description Pts
Size Classification Logic
6 System distinguishes at least two size categories (e.g., small / large) based on ADC threshold; correct category is printed to Serial Monitor or indicated by LED for each graded object.
3–5 Classification logic exists and works for one category consistently, or works for both categories with occasional errors.
1–2 Classification code present but rarely produces the correct result.
0 No classification logic.
 6
LED Visual Feedback
6 LED changes state (color or on/off) to match the classified object size; behavior is consistent and matches the stated design in the Design Brief.
3–5 LED responds but behavior is inconsistent or partially matches design.
1–2 LED is wired and lights up but does not reflect classification.
0 LED does not function.
 6
Section Total/ 12
4 — Slip Detection (8 pts)
Criterion Performance Description Pts
Slip Event Detection
5 When an object slips (or is gently pulled), claw detects the change in ADC reading and responds (re-grips or signals via Serial/LED); demonstrated live.
3–4 Slip detection logic is present and sometimes responds correctly but is unreliable.
1–2 Code contains slip detection logic but no observable response during demo.
0 No slip detection.
 5
Serial / Visual Indicator
3 A distinct Serial message (e.g., "SLIP DETECTED") or LED change occurs at the moment of slip.
2 Indicator fires but is inconsistent or fires at wrong times.
1 Indicator code present but never triggered during demo.
0 No indicator.
 3
Section Total/ 8
5 — Reliability & Code Quality (4 pts)
Criterion Performance Description Pts
Demo Reliability
2 All demonstrated features work on first attempt; no crashes or freezes during the session.
1 One crash or retry required; recovered without instructor help.
0 Multiple failures or instructor intervention needed to recover.

1 Answers one question correctly; struggles with the other.
0 Cannot explain any part of the code to the instructor's satisfaction.
