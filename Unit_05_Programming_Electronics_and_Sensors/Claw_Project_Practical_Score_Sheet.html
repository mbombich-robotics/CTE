Criterion Performance Descriptors Score Notes
1 — Startup & Hardware / 8
Physical Assembly
4 Fully assembled; no loose wires; all components secured.
2–3 Minor issues but functional.
1 Multiple loose connections; sloppy build.
0 Non-functional / not assembled.
/4
Code Uploads & Runs
4 Uploads & runs independently; claw responds within 30 s.
2–3 Runs with one minor prompt (COM port, etc.).
1 Needs significant instructor help to run.
0 Code does not run.
/4
2 — Core Functionality / 18
Servo Control (Open / Close)
6 Opens fully & closes smoothly; motion is controlled (no slamming).
3–5 Opens & closes but inconsistent or limited range.
1–2 Partial movement only (one direction reliable).
0 Servo does not respond.
/6
Potentiometer to Serial
6 Live ADC readings change visibly as claw opens/closes; values sensible (0–1023 or mapped).
3–5 Readings appear but noisy, stuck, or intermittent.
1–2 Serial prints something; values don't reflect position.
0 No Serial output.
/6
Contact Detection
6 Claw stops autonomously on contact; demonstrated on ≥2 objects.
3–5 Works reliably on 1 object, or works on 2 inconsistently.
1–2 Detection logic present but claw overshoots most of the time.
0 No contact detection; stalls or requires manual stop.
/6
3 — Object Classification & LED Feedback / 12
Size Classification Logic
6 ≥2 size categories distinguished by ADC threshold; correct category printed/indicated for each graded object.
3–5 Works for one category consistently, or both with occasional errors.
1–2 Classification code present; rarely correct result.
0 No classification logic.
/6
LED Visual Feedback
6 LED changes (color or on/off) to match classified size; consistent and matches Design Brief.
3–5 LED responds but inconsistent or partially matches design.
1–2 LED wired and lit but doesn't reflect classification.
0 LED does not function.
/6
4 — Slip Detection / 8
Slip Event Detection
5 When object slips/is pulled, claw detects ADC change and responds (re-grips or signals); demonstrated live.
3–4 Logic present; sometimes responds correctly but unreliable.
1–2 Code contains slip logic; no observable response during demo.
0 No slip detection.
/5
Serial / Visual Indicator
3 Distinct Serial message (e.g., "SLIP DETECTED") or LED change occurs at moment of slip.
2 Indicator fires but inconsistent or fires at wrong times.
1 Indicator code present; never triggered during demo.
0 No indicator.
/3
5 — Reliability & Code Quality / 4
Demo Reliability
2 All features work on first attempt; no crashes or freezes.
1 One crash or retry; recovered without instructor help.
0 Multiple failures or instructor intervention needed.
/2
Student Explains Code
2 Answers 1–2 spot questions correctly in own words.
1 Answers one correctly; struggles with the other.
0 Cannot explain any part to instructor's satisfaction.
