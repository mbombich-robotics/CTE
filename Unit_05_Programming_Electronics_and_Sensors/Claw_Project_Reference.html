2. Servo PWM Reference

Pulse Width Range

PositionwriteMicroseconds()
Fully Open500 µs
Center~1450 µs
Fully Closed2400 µs

This is an extended range (500–2400 µs). Standard servos use 1000–2000 µs. Use writeMicroseconds(), not write().

Basic Setup Code

#include <Servo.h> #include <WiFiNINA.h> #define GRIPPER_SERVO 19 #define GRIPPER_FBK A0 Servo claw; void setup() { claw.attach(GRIPPER_SERVO, 500, 2400); claw.writeMicroseconds(500); // open }

What PWM Actually Does

A servo is controlled by sending pulses on a regular interval (every ~20ms). The width of each pulse (in microseconds) tells the servo where to go:

  • Short pulse (500 µs) → servo rotates toward open position
  • Long pulse (2400 µs) → servo rotates toward closed position
  • The servo holds that position as long as pulses keep coming

The Servo library handles the timing — writeMicroseconds() just tells it which width to use.

Closing Loop Pattern

// Close slowly in steps of 15µs for (int pos = 500; pos <= 2400; pos += 15) { claw.writeMicroseconds(pos); delay(25); // check feedback here }
3. Feedback & ADC Reference

What the Feedback Wire Does

The Pololu claw has a built-in potentiometer that outputs a voltage proportional to jaw position:

Jaw PositionVoltageADC Value (approx)
Fully Open~0.5 V~155
Midway~1.35 V~419
Fully Closed~2.2 V~682

Your actual values may differ — calibrate with Serial Monitor on Day 3.

Reading Feedback

int fb = analogRead(GRIPPER_FBK); // returns 0–1023 // To convert to voltage: float v = (fb / 1023.0) * 3.3; Serial.print(fb); Serial.print(" → "); Serial.println(v);

The ADC Formula

The RP2040 Connect uses a 3.3V reference with 10-bit resolution.

ADC = (voltage / 3.3) × 1023

Example: 0.5V open position:

(0.5 / 3.3) × 1023 = 155

⚠️ This board uses 3.3V, not 5V like a standard Arduino. This changes all your expected ADC values — document this in your Key Concepts section.

Contact Detection Logic

While closing, feedback should increase each step. If it stops increasing, the servo has stalled against an object:

int current = analogRead(GRIPPER_FBK); int delta = current - prev; if (delta < 5 && pos > 700) { // contact detected — break contactFb = current; break; } prev = current;

pos > 700 prevents false triggering at the very start of the loop.

4. Full Algorithm Flowchart
[ Serial trigger received ] ↓ [ Open claw → delay 600ms ] ↓ ← place object here [ prev = analogRead(GRIPPER_FBK) ] ↓ [ for pos = 500 → 2400, step 15 ] ↓ [ writeMicroseconds(pos) · delay 25ms ] ↓ [ current = analogRead(A0) ] ↓ [ current − prev < 5 AND pos > 700 ? ] YES → contact! break loop NO → prev = current, continue ↓ [ contactFb < 300 ? ] YES → LED GREEN (large) [ contactFb < 500 ? ] YES → LED BLUE (medium) NO → LED RED (small) ↓ [ monitorSlip(contactFb) — hold 5s, watch for drift > 15 ] ↓ [ if drift detected → flash YELLOW ] ↓ [ open claw · LED off ]

5. AI Prompting Guide

Rule 1 — Give Context

Tell AI your hardware, your knowledge level, and your goal. Vague prompts give vague answers.

✗ Bad:

"How do I read analog input?"

✓ Better:

"I'm using an Arduino Nano RP2040 Connect with a 3.3V reference. I want to read a voltage between 0.5V and 2.2V on pin A0 using analogRead(). Show me the code and explain what the returned number means."

Rule 2 — Show Your Thinking First

Share your plan or pseudocode before asking AI to write code. AI improves your thinking — it doesn't replace it.

✓ Good:

"My plan: close the servo in a loop and check the feedback each step. If feedback stops changing, the servo has stalled. Here's my pseudocode: [paste it]. Translate this to Arduino C++."

Rule 3 — Push Back

AI makes mistakes. If something doesn't make sense, say so. Good example:

"You said the max ADC value is 1024, but earlier you said 1023. Which is correct and why?"
