What is Line Following?

The Concept

  • Robot autonomously follows a black line on a white surface
  • Uses IR sensors to detect the line
  • Adjusts motor speeds to stay on track
  • No human control needed once started

Real-World Applications

  • Warehouse robots following paths
  • Automated guided vehicles (AGVs)
  • Manufacturing assembly lines
  • Airport luggage systems

Bang-Bang Control Explained

What is Bang-Bang Control?

Bang-bang control (also called on-off control) is the simplest form of feedback control. It makes binary decisions - there's no middle ground.

Bang-Bang Characteristics

  • Binary decisions: Turn OR don't turn
  • No proportional control: Same correction every time
  • Simple to program: Just if-else statements
  • Fast response: Immediate corrections
Drawback: Creates zigzag motion. Robot oscillates around the line rather than following it smoothly.

Example: Home Thermostat

Too cold?
↓ YES
Turn heater ON (full power)
↓
Too hot?
↓ YES
Turn heater OFF (no power)

No "medium heat" - it's either ON or OFF!

3-Sensor Basic Approach

Using Only Three Sensors

For basic line following, we only need sensors 2, 3, and 4 (left, center, right).

Sensor Pattern S2 (Left) S3 (Center) S4 (Right) Meaning Action
1 0 1 HIGH LOW HIGH Line Centered Go Straight
0 1 1 LOW HIGH HIGH Line on Left Turn Left
0 0 1 LOW LOW HIGH Line Far Left Turn Left
1 1 0 HIGH HIGH LOW Line on Right Turn Right
1 0 0 HIGH LOW LOW Line Far Right Turn Right
1 1 1 HIGH HIGH HIGH No Line Stop/Search
Note: These are the most common patterns. Some patterns may not occur depending on line width.

Reading the Center Sensor

The Most Important Rule

If the center sensor detects the line (LOW), the robot should go straight.

// Read center sensor (S3 on pin 8) int centerSensor = digitalRead(SENSOR_3); if (centerSensor == LOW) { // Center sensor sees the black line // Robot is centered - go straight! moveForward(150); } else { // Center sensor does NOT see the line // Need to check left and right sensors }
Center Sensor Logic
Is center sensor LOW?
↓ YES
Robot is on track!
Go straight forward
↓ NO
Robot is off-center
Check left/right sensors

Turning Logic

Left and Right Sensor Rules

  • If left sensor sees line (LOW) → Turn LEFT
  • If right sensor sees line (LOW) → Turn RIGHT
  • Simple and intuitive!
// Read left and right sensors int leftSensor = digitalRead(SENSOR_2); // Pin 7 int rightSensor = digitalRead(SENSOR_4); // Pin 9 int centerSensor = digitalRead(SENSOR_3); // Pin 8 if (centerSensor == LOW) { // Line is centered - go straight moveForward(150); } else if (leftSensor == LOW) { // Line is on the left - turn left to get back on track turnLeft(); } else if (rightSensor == LOW) { // Line is on the right - turn right to get back on track turnRight(); } else { // No sensors see the line - we're lost! stopMotors(); }
Important: The order of if-else statements matters! We check center first because it's the ideal case.

Complete Bang-Bang Algorithm

↓
S3 == LOW?
(center on line)
YES ✓
↓

NO ✗
↓
S2 == LOW?

YES

NO
S4 == LOW?
YES: turnRight()

Algorithm Summary

  1. Center sensor on line? Go straight
  2. Else, left sensor on line? Turn left
  3. Else, right sensor on line? Turn right
  4. Else (no sensors on line)? Stop

Full Working Example Code

// 3-Sensor Line Following Robot // Motor pins const int LEFT_MOTOR_PIN1 = 2; const int LEFT_MOTOR_PIN2 = 3; const int RIGHT_MOTOR_PIN1 = 4; const int RIGHT_MOTOR_PIN2 = 5; // Sensor pins (using 3 sensors) const int SENSOR_LEFT = 7; // S2 const int SENSOR_CENTER = 8; // S3 const int SENSOR_RIGHT = 9; // S4 void setup() { // Motor pins as outputs pinMode(LEFT_MOTOR_PIN1, OUTPUT); pinMode(LEFT_MOTOR_PIN2, OUTPUT); pinMode(RIGHT_MOTOR_PIN1, OUTPUT); pinMode(RIGHT_MOTOR_PIN2, OUTPUT); // Sensor pins as inputs pinMode(SENSOR_LEFT, INPUT); pinMode(SENSOR_CENTER, INPUT); pinMode(SENSOR_RIGHT, INPUT); } void loop() { // Read sensors int left = digitalRead(SENSOR_LEFT); int center = digitalRead(SENSOR_CENTER); int right = digitalRead(SENSOR_RIGHT); // Decision logic if (center == LOW) { // Line centered - go straight moveForward(150); } else if (left == LOW) { // Line on left - turn left turnLeft(); } else if (right == LOW) { // Line on right - turn right turnRight(); } else { // No line detected - stop stopMotors(); } } // Motor control functions (from Lesson 6) void moveForward(int speed) { analogWrite(LEFT_MOTOR_PIN1, speed); digitalWrite(LEFT_MOTOR_PIN2, LOW); analogWrite(RIGHT_MOTOR_PIN1, speed); digitalWrite(RIGHT_MOTOR_PIN2, LOW); } void turnLeft() { digitalWrite(LEFT_MOTOR_PIN1, LOW); digitalWrite(LEFT_MOTOR_PIN2, HIGH); digitalWrite(RIGHT_MOTOR_PIN1, HIGH); digitalWrite(RIGHT_MOTOR_PIN2, LOW); } void turnRight() { digitalWrite(LEFT_MOTOR_PIN1, HIGH); digitalWrite(LEFT_MOTOR_PIN2, LOW); digitalWrite(RIGHT_MOTOR_PIN1, LOW); digitalWrite(RIGHT_MOTOR_PIN2, HIGH); } void stopMotors() { digitalWrite(LEFT_MOTOR_PIN1, LOW); digitalWrite(LEFT_MOTOR_PIN2, LOW); digitalWrite(RIGHT_MOTOR_PIN1, LOW); digitalWrite(RIGHT_MOTOR_PIN2, LOW); }

Improved 5-Sensor Logic

Why Use All 5 Sensors?

  • Better detection of sharp curves
  • Earlier warning when line is drifting far off
  • More accurate positioning information
  • Can distinguish between slight and sharp turns
// Enhanced 5-sensor line following void loop() { // Read all 5 sensors int s1 = digitalRead(SENSOR_1); // Far left int s2 = digitalRead(SENSOR_2); // Left int s3 = digitalRead(SENSOR_3); // Center int s4 = digitalRead(SENSOR_4); // Right int s5 = digitalRead(SENSOR_5); // Far right // Centered - go straight if (s3 == LOW) { moveForward(150); } // Slight left - gentle correction else if (s2 == LOW && s1 == HIGH) { turnLeft(); } // Far left - sharp turn needed else if (s1 == LOW) { turnLeft(); // Could use faster turn speed here } // Slight right - gentle correction else if (s4 == LOW && s5 == HIGH) { turnRight(); } // Far right - sharp turn needed else if (s5 == LOW) { turnRight(); // Could use faster turn speed here } // No line detected else { stopMotors(); } }

Advanced Improvement Ideas

  • Use different turn speeds for slight vs. sharp corrections
  • Reduce forward speed during turns
  • Add search behavior when line is lost
  • Remember last known direction and search that way

Alternative Line Following Code

About This Code

This is a more advanced line following program that uses a speed lookup table approach. Instead of simple if/else decisions, it maps each sensor pattern to specific left and right motor speeds stored in an array.

  • speed_array[2][10] — A 2D array that holds motor speeds for each sensor position
  • DEBUG mode — Enabled by default; set to 0 to run normally
  • PWM ramping — Gradually changes speed to protect motors
  • 5-sensor IR array — Uses all 5 sensors for smoother line detection

How to Use This Code

  1. Upload the code and open Serial Monitor (9600 baud)
  2. With #define DEBUG 1, the robot will run at fixed PWM values — use this to verify your motor wiring
  3. Set #define DEBUG 0 to enable normal line following
  4. Fill in your calibrated speed values in the speed_array table
  5. Test on your track and adjust speeds as needed

Exercise 4: Speed Optimization (10 min)

  1. Experiment with different forward speeds (100, 150, 200)
  2. Does faster = better? Or does it make control harder?
  3. Try reducing speed during turns
  4. Find the fastest speed that still follows reliably

Common Problems and Solutions

Problem Possible Solution
Robot goes straight off the line Check sensor readings - are they inverted? Verify LOW = black line
Robot spins in circles Verify motor wiring - left/right may be swapped
Robot oscillates wildly Reduce motor speed for gentler corrections
Robot can't follow curves Line may be too sharp - use wider curves OR reduce speed
Sensors not detecting line Check sensor height (2-10mm optimal) and line contrast

Testing Tips

  • Start with a straight line before trying curves
  • Use electrical tape for best contrast
  • Test in consistent lighting conditions
  • Add Serial.println() to debug sensor readings
  • Make ONE change at a time when troubleshooting
