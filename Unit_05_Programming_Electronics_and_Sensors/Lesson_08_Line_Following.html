What is Line Following?

The Concept

  • Robot autonomously follows a black line on a white surface
  • Uses IR sensors to detect the line
  • Adjusts motor speeds to stay on track
  • No human control needed once started

Real-World Applications

  • Warehouse robots following paths
  • Automated guided vehicles (AGVs)
  • Manufacturing assembly lines
  • Airport luggage systems

Line Width: Typically 15-25mm wide. Must be narrower than sensor array width.

Bang-Bang Control Explained

What is Bang-Bang Control?

Bang-bang control (also called on-off control) is the simplest form of feedback control. It makes binary decisions - there's no middle ground.

Bang-Bang Characteristics

  • Binary decisions: Turn OR don't turn
  • No proportional control: Same correction every time
  • Simple to program: Just if-else statements
  • Fast response: Immediate corrections
Drawback: Creates zigzag motion. Robot oscillates around the line rather than following it smoothly.

Example: Home Thermostat

Too cold?
↓ YES
Turn heater ON (full power)
↓
Too hot?
↓ YES
Turn heater OFF (no power)

No "medium heat" - it's either ON or OFF!

3-Sensor Basic Approach

Using Only Three Sensors

For basic line following, we only need sensors 2, 3, and 4 (left, center, right).

Sensor Pattern S2 (Left) S3 (Center) S4 (Right) Meaning Action
1 0 1 HIGH LOW HIGH Line Centered Go Straight
0 1 1 LOW HIGH HIGH Line Far Left Turn Left sharply
0 0 1 LOW LOW HIGH Line Slightly Left Turn Left
1 1 0 HIGH HIGH LOW Line Far Right Turn Right sharply
1 0 0 HIGH LOW LOW Line Slightly Right Turn Right
1 1 1 HIGH HIGH HIGH No Line Stop/Search
Note: These are the most common patterns. Some patterns may not occur depending on line width.

Reading the Center Sensor

The Most Important Rule

If the center sensor detects the line (LOW), the robot should go straight.

// Read center sensor (S3 on pin 2) int centerSensor = digitalRead(SENSOR_3); if (centerSensor == LOW) { // Center sensor sees the black line // Robot is centered - go straight! moveForward(150); } else { // Center sensor does NOT see the line // Need to check left and right sensors }
Center Sensor Logic
Is center sensor LOW?
↓ YES
Robot is on track!
Go straight forward
↓ NO
Robot is off-center
Check left/right sensors

Turning Logic

Left and Right Sensor Rules

  • If left sensor sees line (LOW) → Turn LEFT
  • If right sensor sees line (LOW) → Turn RIGHT
  • Simple and intuitive!
// Read left and right sensors int leftSensor = digitalRead(SENSOR_2); // Pin 0 int rightSensor = digitalRead(SENSOR_4); // Pin 1 int centerSensor = digitalRead(SENSOR_3); // Pin 2 if (centerSensor == LOW) { // Line is centered - go straight moveForward(150); } else if (leftSensor == LOW) { // Line is on the left - turn left to get back on track turnLeft(); } else if (rightSensor == LOW) { // Line is on the right - turn right to get back on track turnRight(); } else { // No sensors see the line - we're lost! stopMotors(); }
Important: The order of if-else statements matters! We check center first because it's the ideal case.

Algorithm Summary

  1. Center sensor on line? Go straight
  2. Else, left sensor on line? Turn left
  3. Else, right sensor on line? Turn right
  4. Else (no sensors on line)? Stop

Full Working Example Code

// 3-Sensor Line Following Robot // Motor pins - 3-pin motor controller (INA, INB, PWM) const int LEFT_MOTOR_INA = 8; const int LEFT_MOTOR_INB = 7; const int LEFT_MOTOR_PWM = 5; const int RIGHT_MOTOR_INA = 6; const int RIGHT_MOTOR_INB = 4; const int RIGHT_MOTOR_PWM = 3; // Sensor pins (using 3 sensors) const int SENSOR_LEFT = 0; // S2 const int SENSOR_CENTER = 2; // S3 const int SENSOR_RIGHT = 1; // S4 void setup() { // Motor pins as outputs pinMode(LEFT_MOTOR_INA, OUTPUT); pinMode(LEFT_MOTOR_INB, OUTPUT); pinMode(LEFT_MOTOR_PWM, OUTPUT); pinMode(RIGHT_MOTOR_INA, OUTPUT); pinMode(RIGHT_MOTOR_INB, OUTPUT); pinMode(RIGHT_MOTOR_PWM, OUTPUT); // Sensor pins as inputs pinMode(SENSOR_LEFT, INPUT); pinMode(SENSOR_CENTER, INPUT); pinMode(SENSOR_RIGHT, INPUT); } void loop() { // Read sensors int left = digitalRead(SENSOR_LEFT); int center = digitalRead(SENSOR_CENTER); int right = digitalRead(SENSOR_RIGHT); // Decision logic if (center == LOW) { // Line centered - go straight moveForward(150); } else if (left == LOW) { // Line on left - turn left turnLeft(); } else if (right == LOW) { // Line on right - turn right turnRight(); } else { // No line detected - stop stopMotors(); } } // Motor control functions (from Lesson 6) void moveForward(int speed) { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, speed); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, speed); } void turnLeft() { digitalWrite(LEFT_MOTOR_INA, HIGH); digitalWrite(LEFT_MOTOR_INB, LOW); analogWrite(LEFT_MOTOR_PWM, 150); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, 150); } void turnRight() { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, 150); digitalWrite(RIGHT_MOTOR_INA, HIGH); digitalWrite(RIGHT_MOTOR_INB, LOW); analogWrite(RIGHT_MOTOR_PWM, 150); } void stopMotors() { analogWrite(LEFT_MOTOR_PWM, 0); analogWrite(RIGHT_MOTOR_PWM, 0); digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, LOW); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, LOW); }

Improved 5-Sensor Logic

Why Use All 5 Sensors?

  • Better detection of sharp curves
  • Earlier warning when line is drifting far off
  • More accurate positioning information
  • Can distinguish between slight and sharp turns
// Enhanced 5-sensor line following void loop() { // Read all 5 sensors int s1 = digitalRead(SENSOR_1); // Far left int s2 = digitalRead(SENSOR_2); // Left int s3 = digitalRead(SENSOR_3); // Center int s4 = digitalRead(SENSOR_4); // Right int s5 = digitalRead(SENSOR_5); // Far right // Centered - go straight if (s3 == LOW) { moveForward(150); } // Slight left - gentle correction else if (s2 == LOW && s1 == HIGH) { turnLeft(); } // Far left - sharp turn needed else if (s1 == LOW) { turnLeft(); // Could use faster turn speed here } // Slight right - gentle correction else if (s4 == LOW && s5 == HIGH) { turnRight(); } // Far right - sharp turn needed else if (s5 == LOW) { turnRight(); // Could use faster turn speed here } // No line detected else { stopMotors(); } }

Advanced Improvement Ideas

  • Use different turn speeds for slight vs. sharp corrections
  • Reduce forward speed during turns
  • Add search behavior when line is lost
  • Remember last known direction and search that way

Choose Your Approach

Two approaches are provided for line following. Pick the one that fits your experience level.

Basic Approach (This Lesson)

Simple if/else logic with helper functions

  • 3 speed values to tune
  • Works immediately after upload
  • Easy to understand and debug
  • Uses all 5 sensors
  • Standard 8-bit PWM (0-255)
Trade-off: Zigzag motion, less smooth on curves

Advanced Approach (Lesson 9)

Speed lookup table with PWM ramping

  • 18 speed values to tune
  • Requires filling in all values before it moves
  • Smoother, more precise corrections
  • Uses all 5 sensors + in-between positions
  • 12-bit PWM (0-4095) for finer control
Trade-off: More setup time, harder to debug
Recommendation: Start with the Basic approach. Get your robot following the line reliably first. Then upgrade to Advanced in Lesson 9 if you want smoother, faster performance.

Basic Approach: Complete 5-Sensor Code

line_following_basic.ino
// Basic 5-Sensor Line Following // Uses simple if/else logic with gentle and sharp turns // === TUNING VALUES - Change these to tune your robot! === #define FORWARD_SPEED 150 // Speed when going straight (0-255) #define GENTLE_TURN_SPEED 100 // Slow motor speed for gentle turns #define SHARP_TURN_SPEED 150 // Motor speed for sharp turns // Motor pins const int LEFT_MOTOR_INA = 8; const int LEFT_MOTOR_INB = 7; const int LEFT_MOTOR_PWM = 5; const int RIGHT_MOTOR_INA = 6; const int RIGHT_MOTOR_INB = 4; const int RIGHT_MOTOR_PWM = 3; // Sensor pins const int SENSOR_1 = A2; // S1 - Far left const int SENSOR_2 = 0; // S2 - Left const int SENSOR_3 = 2; // S3 - Center const int SENSOR_4 = 1; // S4 - Right const int SENSOR_5 = A3; // S5 - Far right void setup() { pinMode(LEFT_MOTOR_INA, OUTPUT); pinMode(LEFT_MOTOR_INB, OUTPUT); pinMode(LEFT_MOTOR_PWM, OUTPUT); pinMode(RIGHT_MOTOR_INA, OUTPUT); pinMode(RIGHT_MOTOR_INB, OUTPUT); pinMode(RIGHT_MOTOR_PWM, OUTPUT); pinMode(SENSOR_1, INPUT); pinMode(SENSOR_2, INPUT); pinMode(SENSOR_3, INPUT); pinMode(SENSOR_4, INPUT); pinMode(SENSOR_5, INPUT); Serial.begin(9600); } void loop() { int s1 = digitalRead(SENSOR_1); // Far left int s2 = digitalRead(SENSOR_2); // Left int s3 = digitalRead(SENSOR_3); // Center int s4 = digitalRead(SENSOR_4); // Right int s5 = digitalRead(SENSOR_5); // Far right if (s3 == LOW) { moveForward(FORWARD_SPEED); // Centered - go straight } else if (s2 == LOW && s1 == HIGH) { gentleTurnLeft(); // Slightly left - gentle correction } else if (s1 == LOW) { sharpTurnLeft(); // Far left - sharp correction } else if (s4 == LOW && s5 == HIGH) { gentleTurnRight(); // Slightly right - gentle correction } else if (s5 == LOW) { sharpTurnRight(); // Far right - sharp correction } else { stopMotors(); // No line detected - stop } } // Both motors forward at the same speed void moveForward(int speed) { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, speed); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, speed); } // Gentle left: slow left motor, full speed right motor void gentleTurnLeft() { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, GENTLE_TURN_SPEED); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, FORWARD_SPEED); } // Gentle right: full speed left motor, slow right motor void gentleTurnRight() { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, FORWARD_SPEED); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, GENTLE_TURN_SPEED); } // Sharp left: left motor backward, right motor forward void sharpTurnLeft() { digitalWrite(LEFT_MOTOR_INA, HIGH); digitalWrite(LEFT_MOTOR_INB, LOW); analogWrite(LEFT_MOTOR_PWM, SHARP_TURN_SPEED); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, HIGH); analogWrite(RIGHT_MOTOR_PWM, SHARP_TURN_SPEED); } // Sharp right: left motor forward, right motor backward void sharpTurnRight() { digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, HIGH); analogWrite(LEFT_MOTOR_PWM, SHARP_TURN_SPEED); digitalWrite(RIGHT_MOTOR_INA, HIGH); digitalWrite(RIGHT_MOTOR_INB, LOW); analogWrite(RIGHT_MOTOR_PWM, SHARP_TURN_SPEED); } void stopMotors() { analogWrite(LEFT_MOTOR_PWM, 0); analogWrite(RIGHT_MOTOR_PWM, 0); digitalWrite(LEFT_MOTOR_INA, LOW); digitalWrite(LEFT_MOTOR_INB, LOW); digitalWrite(RIGHT_MOTOR_INA, LOW); digitalWrite(RIGHT_MOTOR_INB, LOW); }

Tuning the Basic Code

Step-by-Step Tuning Guide

  1. Verify motor direction: Upload the code. Both motors should spin forward. If one spins backward, swap its INA and INB pin numbers in the code.
  2. Test on a straight line: Place the robot on a straight black line. Does it follow? If it veers off, your left/right sensors or turn functions may be swapped.
  3. Set FORWARD_SPEED: Start at 120. This controls how fast the robot goes on a straight. Higher = faster but harder to control.
  4. Set GENTLE_TURN_SPEED: Start at 80. This is the slow motor speed during gentle corrections (S2/S4). Lower values = gentler turns. The other motor stays at FORWARD_SPEED.
  5. Set SHARP_TURN_SPEED: Start at 130. This controls how aggressively the robot corrects when far off (S1/S5). One motor goes backward at this speed.
  6. Test on curves: If the robot loses the line on curves, either reduce FORWARD_SPEED or increase turn speeds.
  7. Fine-tune: Gradually increase FORWARD_SPEED until the robot can't keep up. Back off slightly. That's your fastest reliable speed!
Key Principle: Change ONE value at a time, then test. If you change multiple values at once, you won't know which change helped (or hurt).

Testing and Troubleshooting

Common Problems and Solutions

Problem Possible Solution
Robot goes straight off the line Check sensor readings - are they inverted? Verify LOW = black line
Robot spins in circles Verify motor wiring - left/right may be swapped
Robot oscillates wildly Reduce motor speed for gentler corrections
Robot can't follow curves Line may be too sharp - use wider curves OR reduce speed
Sensors not detecting line Check sensor height (2-10mm optimal) and line contrast

Testing Tips

  • Start with a straight line before trying curves
  • Use electrical tape for best contrast
  • Test in consistent lighting conditions
  • Add Serial.println() to debug sensor readings
  • Make ONE change at a time when troubleshooting
