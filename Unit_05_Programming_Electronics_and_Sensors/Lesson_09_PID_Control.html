PID Control for Line Following

Smooth, Proportional Control for Superior Performance

What You'll Learn Today

  • Why bang-bang control causes zigzag motion
  • What PID control is and how it works
  • Calculating position error from sensor array
  • Using Proportional (P) control for smooth following
  • Adding Derivative (D) to reduce oscillation
  • Tuning PID constants for optimal performance

Why Do We Need PID Control?

Bang-Bang Control Limitations

  • Binary decisions: Always turns at full power
  • Zigzag motion: Robot oscillates around the line
  • Slow speeds: Must go slow to avoid overshooting
  • Poor efficiency: Constant jerky corrections
Problem: Small errors get the same correction as large errors. This causes overcorrection and oscillation!

PID Control Advantages

  • Proportional response: Small errors = small corrections
  • Smooth motion: Robot follows line gracefully
  • Higher speeds: Can go faster while staying on track
  • Better accuracy: Stays closer to center line
Result: Your robot will follow lines 2-3x faster with smoother, more accurate motion!

What is PID Control?

PID Stands For:

  • P = Proportional (responds to current error)
  • I = Integral (responds to accumulated error over time)
  • D = Derivative (responds to rate of change of error)
correction = Kp × error + Ki × integral + Kd × derivative

For Beginners: Start Simple!

We'll focus on P and D only. The Integral term is rarely needed for line following.

correction = Kp × error + Kd × derivative

What Each Term Does

  • P (Proportional): Main correction - makes robot turn toward line
  • D (Derivative): Dampening - reduces wobbling and overshooting
  • I (Integral): Fixes steady-state error - not needed for our robot!
Learning Approach: First implement P-only control to understand the concept. Then add D to improve performance. This makes tuning much easier!

The Proportional (P) Term

The Key Concept

The correction is proportional to the error. Big error = big correction. Small error = small correction.

correction = Kp × error

Understanding Kp

  • Kp = Proportional constant (you tune this!)
  • Higher Kp = stronger, faster corrections
  • Lower Kp = gentler, slower corrections
  • Typical range: 20 to 50
Too High: Robot oscillates wildly
Too Low: Robot can't correct fast enough
Error Kp = 30 Correction
-2 (far left) 30 -60 (turn left)
-1 (slight left) 30 -30 (gentle left)
0 (centered) 30 0 (straight)
+1 (slight right) 30 +30 (gentle right)
+2 (far right) 30 +60 (turn right)
Key Insight: Notice how the correction scales with the error! This is what makes PID control smooth and efficient.

Using Error to Adjust Motor Speeds

The Motor Speed Formula

Once we have the correction value, we adjust each motor's speed:

Left Motor = baseSpeed - correction
Right Motor = baseSpeed + correction

How It Works

  • Error = 0: Both motors same speed → straight
  • Error < 0: Right motor faster → turn left
  • Error > 0: Left motor faster → turn right

Example: baseSpeed = 150, Kp = 30

Error = -1 (line slightly left)
correction = 30 × (-1) = -30
Left: 150 - (-30) = 180
Right: 150 + (-30) = 120
→ Right motor slows, robot turns left

Important: Always use constrain(speed, 0, 255) to keep motor speeds in valid range! Negative speeds or speeds above 255 will cause problems.

P-Only Control Implementation

Start with Proportional Control Only

Before adding complexity, get P-only control working well. This teaches you the fundamentals!

// P-Only Line Following Controller // PID Constants const float Kp = 30.0; // Proportional constant (tune this!) const int baseSpeed = 150; // Base motor speed void loop() { // Step 1: Calculate position error from sensors int error = getPositionError(); // Step 2: Calculate P correction float correction = Kp * error; // Step 3: Adjust motor speeds int leftSpeed = baseSpeed - correction; int rightSpeed = baseSpeed + correction; // Step 4: Constrain to valid range (0-255) leftSpeed = constrain(leftSpeed, 0, 255); rightSpeed = constrain(rightSpeed, 0, 255); // Step 5: Apply to motors setMotorSpeeds(leftSpeed, rightSpeed); } // Returns position error based on sensor readings // Negative = line is left, Positive = line is right int getPositionError() { // Read all 5 sensors (LOW = black line detected) int s1 = digitalRead(SENSOR_1); int s2 = digitalRead(SENSOR_2); int s3 = digitalRead(SENSOR_3); int s4 = digitalRead(SENSOR_4); int s5 = digitalRead(SENSOR_5); // Simple position calculation if (s1 == LOW) return -2; // Far left else if (s2 == LOW) return -1; // Left else if (s3 == LOW) return 0; // Center else if (s4 == LOW) return 1; // Right else if (s5 == LOW) return 2; // Far right return 0; // Default: assume centered if no sensors detect line }

What You'll Notice

  • Much smoother motion than bang-bang control
  • Robot makes small adjustments continuously
  • May still oscillate slightly - that's where D comes in!

The Derivative (D) Term

Why Add Derivative?

The D term responds to how fast the error is changing. It helps prevent overshooting and reduces wobbling.

derivative = error - lastError

What D Does

  • Dampens oscillation: Reduces back-and-forth wobbling
  • Predicts future error: Starts correcting before error gets large
  • Smooths response: Makes corrections more gradual

Understanding Kd

  • Kd = Derivative constant
  • Higher Kd = more dampening
  • Typical range: 5 to 20
  • Start with Kd = Kp / 3
How D Helps
Scenario: Robot crossing center line
Last error: -1 (line was left)
Current error: +1 (line now right)
Derivative: +1 - (-1) = +2
This large positive derivative tells the robot:
"Whoa! You're turning too fast left!"
The D term adds a counteracting force to slow down the turn.
Result: Robot smoothly approaches center instead of overshooting and oscillating.
Tuning Tip: If Kd is too high, robot becomes sluggish and responds slowly. If too low, you still get oscillation. Find the sweet spot!

Complete P+D Controller Example

// Full PD Line Following Controller // PID Constants (tune these!) const float Kp = 30.0; // Proportional constant const float Kd = 10.0; // Derivative constant const int baseSpeed = 150; // Base motor speed // Variables to track error over time int lastError = 0; // Previous error for derivative void loop() { // Step 1: Read sensors and calculate current error int error = getPositionError(); // Step 2: Calculate derivative (rate of change of error) int derivative = error - lastError; // Step 3: Calculate total correction (P + D) float correction = (Kp * error) + (Kd * derivative); // Step 4: Calculate motor speeds int leftSpeed = baseSpeed - correction; int rightSpeed = baseSpeed + correction; // Step 5: Constrain to valid range leftSpeed = constrain(leftSpeed, 0, 255); rightSpeed = constrain(rightSpeed, 0, 255); // Step 6: Apply to motors setMotorSpeeds(leftSpeed, rightSpeed); // Step 7: Save current error for next iteration lastError = error; } // Motor control function using analogWrite for variable speed void setMotorSpeeds(int leftSpeed, int rightSpeed) { // Left motor if (leftSpeed >= 0) { analogWrite(LEFT_MOTOR_PIN1, leftSpeed); digitalWrite(LEFT_MOTOR_PIN2, LOW); } else { digitalWrite(LEFT_MOTOR_PIN1, LOW); analogWrite(LEFT_MOTOR_PIN2, -leftSpeed); } // Right motor if (rightSpeed >= 0) { analogWrite(RIGHT_MOTOR_PIN1, rightSpeed); digitalWrite(RIGHT_MOTOR_PIN2, LOW); } else { digitalWrite(RIGHT_MOTOR_PIN1, LOW); analogWrite(RIGHT_MOTOR_PIN2, -rightSpeed); } } // Get position error from sensors int getPositionError() { int s1 = digitalRead(SENSOR_1); int s2 = digitalRead(SENSOR_2); int s3 = digitalRead(SENSOR_3); int s4 = digitalRead(SENSOR_4); int s5 = digitalRead(SENSOR_5); // Return position based on which sensor(s) see the line if (s1 == LOW) return -2; else if (s2 == LOW) return -1; else if (s3 == LOW) return 0; else if (s4 == LOW) return 1; else if (s5 == LOW) return 2; return lastError; // If no line detected, maintain last known position }

Key Points

  • Notice how we save lastError at the end of each loop
  • The derivative is simply the change in error since last reading
  • Both P and D terms are added together for total correction
  • Always constrain motor speeds to prevent invalid values

Tuning PID Constants

Step-by-Step Tuning Process

Tuning is the process of finding the best Kp and Kd values for your robot. It requires experimentation!

PID Tuning Workflow
STEP 1: Set Kd = 0 (disable derivative)

Common Symptoms

  • Kp too low: Slow response, can't make sharp turns
  • Kp too high: Wild oscillation, unstable
  • Kd too low: Some wobbling remains
  • Kd too high: Sluggish, slow to respond

Starting Values

Parameter Recommended Start
baseSpeed 150
Kp 30
Kd 10

Pro Tips

  • Test on the same track section for consistency
  • Make ONE change at a time
  • Write down all values you test
  • Once tuned, you can increase baseSpeed for faster following!

Advanced Topics and Next Steps

Weighted Sensor Reading (Advanced)

For even better accuracy, use weighted average when multiple sensors detect the line:

int getPositionError() { int s1 = digitalRead(SENSOR_1) == LOW ? 1 : 0; int s2 = digitalRead(SENSOR_2) == LOW ? 1 : 0; int s3 = digitalRead(SENSOR_3) == LOW ? 1 : 0; int s4 = digitalRead(SENSOR_4) == LOW ? 1 : 0; int s5 = digitalRead(SENSOR_5) == LOW ? 1 : 0; int sum = s1 + s2 + s3 + s4 + s5; if (sum == 0) return lastError; // No line detected int weightedSum = (-2*s1) + (-1*s2) + (0*s3) + (1*s4) + (2*s5); return weightedSum / sum; // Average position }

When to Use Full PID

The Integral term is useful for:

  • Robots with systematic bias (one motor weaker)
  • Following on slopes/inclines
  • Eliminating steady-state error

For most line followers, PD is sufficient!

Real-World PID Applications

  • Cruise control: Maintains constant vehicle speed
  • Quadcopters: Stabilizes flight and maintains altitude
  • 3D printers: Controls temperature of hotend and bed
  • Industrial robots: Precise position control of robot arms
  • HVAC systems: Temperature regulation in buildings
