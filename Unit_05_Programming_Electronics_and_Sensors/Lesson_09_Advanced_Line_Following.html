Advanced Line Following

PD Control — Proportional + Derivative

What You'll Learn Today

  • How position error gives a number for how far off the line we are
  • The Proportional (P) term — turn harder when further off
  • The Derivative (D) term — reduce oscillation

From Basic to Advanced

Lesson 8: Basic Approach

  • Simple if/else checks: if (s3 == LOW)
  • Fixed responses: turnLeft() or turnRight()
  • Same turn speed whether slightly off or way off
  • Results in jerky, oscillating movement

Lesson 9: PD Approach

  • Calculate a position error number (-2 to +2)
  • Proportional correction based on how far off
  • Derivative dampens oscillation
  • Results in smooth, fast line following

The Key Idea

Instead of "turn left" or "turn right," we calculate how much to turn based on how far off the line we are. A small error gets a small correction. A big error gets a big correction.

Understanding Position Error

Our 5 IR sensors give us a number from -2 to +2 that tells us exactly where the line is relative to the robot:

Sensor Detecting Line Error Value Meaning Action Needed
S1 (Far Left) -2 Line is far to the left Hard left turn
S2 (Left) -1 Line is slightly left Gentle left turn
S3 (Center) 0 Perfectly centered! Go straight
S4 (Right) +1 Line is slightly right Gentle right turn
S5 (Far Right) +2 Line is far to the right Hard right turn

Proportional Control (P)

The P Term

The proportional term creates a correction that is proportional to the error. Bigger error = bigger correction.

correction = Kp × error
ErrorKp = 30Correction
-230 × -2-60
-130 × -1-30
030 × 00
+130 × +1+30
+230 × +2+60

Applying the Correction

Left motor = baseSpeed − correction
Right motor = baseSpeed + correction

If line is to the right (error = +1), correction is positive: left motor speeds up, right motor slows down → robot turns right toward the line.

Kp is the "gain" — it controls how aggressively the robot responds. Too low = sluggish. Too high = oscillation. Start with Kp = 30 and adjust.

The Problem with P-Only Control

Robot path with P-only control (high Kp):

  Start                                    Goal
    |                                        |
    |   >>>>>>>\                              |
    |            \   />>>>\                  |
    |             \ /      \   />>>>\       |
====|==============X========\ /======\======|====  ← LINE
    |                        X        \>>>> |
    |                                        |

The robot overshoots the line, then corrects back, then overshoots again!

Why It Oscillates

  • Robot is off to the right → P says "turn left!"
  • Robot crosses the center → P still says "turn left!" (for one more read)
  • Now off to the left → P says "turn right!"
  • Repeats endlessly — oscillation

The Solution: Add D

  • The Derivative term looks at how fast the error is changing
  • If error is shrinking quickly, D says "ease off!"
  • Acts like a brake on the correction
  • Prevents overshoot → smooth following

Adding the Derivative (D)

What is the Derivative?

The derivative is the rate of change of the error — how fast the error is changing from one loop to the next.

derivative = error − lastError
Previous Error Current Error Derivative What It Means
+2 +1 -1 Getting better! Error is shrinking → ease off
0 +1 +1 Getting worse! Error is growing → correct harder
+1 +1 0 No change → D term has no effect
-1 +1 +2 Crossed the line! Strong damping needed
correction = (Kp × error) + (Kd × derivative)

Start with Kd = Kp / 3 (e.g., if Kp = 30, try Kd = 10). Kd too high = sluggish robot. Kd too low = still oscillating.

Motor Calibration

The Problem: Uneven Motors

Even at the same PWM value, your two motors likely spin at different speeds. This causes the robot to drift to one side even when it should go straight.

The Solution: Speed Coefficient

We measure each motor's actual speed and apply a correction factor to the faster motor so both motors match.

motorCoefficient = slowerSpeed / fasterSpeed

Example: If left motor goes 180 RPM and right goes 200 RPM at PWM 150:
rightCoefficient = 180/200 = 0.90

How to Calibrate

  1. Upload the Motor Tuning Sketch
  2. Set both motors to the same PWM
  3. Mark a spot on each wheel
  4. Count rotations in 10 seconds
  5. Calculate the coefficient
  6. Enter it in your line follower code
// In your line follower code, apply the coefficient: const float LEFT_COEFF = 1.0; // Adjust if left motor is faster const float RIGHT_COEFF = 0.90; // Example: right motor is 10% faster analogWrite(LEFT_MOTOR_PWM, leftSpeed * LEFT_COEFF); analogWrite(RIGHT_MOTOR_PWM, rightSpeed * RIGHT_COEFF);

Lab Exercises

Exercise 1: Motor Calibration

  1. Upload the Motor Tuning Sketch (separate file)
  2. Run both motors at the same PWM (e.g., 150)
  3. Count wheel rotations for each motor over 10 seconds
  4. Calculate the coefficient and enter it in your PD code

Exercise 2: P-Only Control

  1. Set Kd = 0 at the top of the starter code
  2. Complete the TODOs in loop() (skip the Kd term for now)
  3. Upload and test — robot should follow but will oscillate

Exercise 3: Tune Kp

  1. Try different Kp values: 20, 30, 40, 50
  2. Find the highest value that doesn't oscillate too badly
  3. Write down your best Kp value!

Lab Exercises (continued)

Exercise 4: Add the D Term

  1. Set Kd = Kp / 3
  2. Complete the full PD calculation in the correction formula
  3. Upload and test — oscillation should be reduced!

Exercise 5: Fine-tune Kd

  1. Adjust Kd up or down by 2-3 at a time
  2. If still wobbling → increase Kd
  3. If sluggish → decrease Kd
  4. Find the smoothest possible line following

Exercise 6: Optimize for Speed

  1. Once tuned, increase baseSpeed gradually: 150 → 170 → 190 → 200
  2. You may need to re-tune Kp and Kd for higher speeds
  3. Find your robot's maximum reliable speed!

Tuning Tips & Troubleshooting

Symptom Likely Cause Solution
Robot barely turns, can't follow curves Kp too low Increase Kp
Robot oscillates wildly side-to-side Kp too high, no D term Decrease Kp, or add/increase Kd
Robot follows but wobbles slightly Needs more damping Increase Kd
Robot responds slowly, sluggish Kd too high Decrease Kd or increase Kp
Robot leaves the line on sharp curves Too fast or low Kp Reduce baseSpeed or increase Kp
Robot drifts to one side on straight sections Motors not calibrated Run Motor Tuning Sketch and adjust coefficients

Wrap Up

What You Learned

  • Position error converts sensor readings to a number (-2 to +2)
  • P term (Kp × error) gives proportional correction
  • D term (Kd × derivative) reduces oscillation
  • Motor calibration ensures both motors drive evenly
  • Tuning is key — change ONE variable at a time!
