Analog vs Digital

Digital (Yesterday)

Only TWO states:

  • HIGH (5V) or LOW (0V)
  • Like a light switch: ON/OFF
  • Perfect for buttons, switches

Analog (Today)

RANGE of values:

  • 0-1023 (10-bit resolution)
  • Like a dimmer switch: brightness levels
  • Perfect for sensors, potentiometers
🔍 Deep Dive: Why 1023? ▼

Arduino's ADC (Analog-to-Digital Converter) is 10-bit. This means it can represent 2¹⁰ = 1024 different values. Since we start counting at 0, the range is 0-1023. It measures voltage from 0V to 5V and divides it into 1024 steps. So each step is about 4.9mV (5V ÷ 1024).

Reading Analog Values

int potPin = A0; // Analog pin A0 int potValue = 0; // Variable to store reading void setup() { Serial.begin(9600); // No pinMode needed for analog inputs! } void loop() { potValue = analogRead(potPin); // Read 0-1023 Serial.print("Pot Value: "); Serial.println(potValue); delay(100); }

Key Points

  • analogRead() always returns 0-1023
  • Use pins A0-A5 for analog input
  • No pinMode() needed for analog pins
  • Reads voltage: 0V = 0, 5V = 1023

💡 Converting to Voltage ▼

To convert the reading to voltage:

float voltage = (potValue / 1023.0) * 5.0; Serial.println(voltage);

Or use map() for integer math (faster)!

PWM: Pulse Width Modulation

PWM creates the illusion of analog output by rapidly switching digital HIGH/LOW.

  • 25% duty cycle = dim LED
  • 50% duty cycle = medium brightness
  • 75% duty cycle = bright LED
  • Switches so fast (~490 Hz) you can't see it

PWM Pins on Arduino Uno

Only certain pins support PWM (marked with ~):

Pins: 3, 5, 6, 9, 10, 11

analogWrite() - PWM Output

int ledPin = 9; // PWM pin int brightness = 0; // 0-255 void setup() { pinMode(ledPin, OUTPUT); } void loop() { analogWrite(ledPin, brightness); // Set brightness brightness = brightness + 5; // Increase if (brightness > 255) { brightness = 0; // Reset } delay(30); // Smooth fade }

Key Points

  • analogWrite(pin, value) takes 0-255
  • 0 = completely off, 255 = completely on
  • Only works on PWM pins (~3, ~5, ~6, ~9, ~10, ~11)
  • Still need pinMode(pin, OUTPUT)

The map() Function

Problem: analogRead() gives 0-1023, but analogWrite() needs 0-255!

int potValue = analogRead(A0); // 0-1023 int ledBrightness = map(potValue, 0, 1023, 0, 255); analogWrite(9, ledBrightness); // 0-255

map() Syntax

map(value, fromLow, fromHigh, toLow, toHigh)
  • value - the number to map
  • fromLow/fromHigh - input range
  • toLow/toHigh - output range
🔍 Deep Dive: How map() Works ▼

map() uses proportional scaling:

// If potValue = 512 (middle of 0-1023) // map(512, 0, 1023, 0, 255) returns: // (512 - 0) * (255 - 0) / (1023 - 0) + 0 = 127.5 ≈ 127

It maintains the same position in the new range!

⚠️ The Problems:

  • Bug 1: When i=10, motorSpeed = 250 (OK), but then analogWrite(9, 300) happens!
  • Bug 2: PWM values MUST be 0-255. Using 300 is undefined behavior!

✓ The Fix:

  • Bug 1: The last motorSpeed value should be 250, not go to 300
  • Bug 2: Change analogWrite(9, 300) to analogWrite(9, 255)
  • Remember: PWM range is ALWAYS 0-255, never higher!
