What is a Microcontroller?

A complete computer on a single chip

Your Computer (PC/Laptop)

  • General purpose
  • Multiple programs at once
  • Gigabytes of RAM
  • Complex operating system
  • Keyboard, mouse, display

A Microcontroller

  • Dedicated purpose
  • One program at a time
  • Kilobytes of RAM
  • Simple or no operating system
  • Direct hardware control

Microcontrollers are EVERYWHERE: Cars, microwaves, thermostats, toys, medical devices, washing machines, your robot!

Microcontroller Components

Understanding Each Component

CPU (Central Processing Unit)

The "brain" that executes your code instructions one at a time, very fast (16 MHz = 16 million instructions/second!)

Flash Memory

Stores your program permanently. Even when powered off, your code stays here. Arduino Uno: 32 KB (internal). Your RP2040: 16 MB (external chip on board)

RAM (Random Access Memory)

Temporary storage for variables while running. Lost when powered off. Arduino Uno: 2 KB (tiny!)

I/O Pins (Input/Output)

Connect to the real world: read sensors, control motors, light LEDs. This is how code becomes action!

Key Takeaway: Your code runs on the CPU, uses RAM for variables, and controls I/O pins to interact with hardware

The Arduino Board

An Arduino is a development board that makes microcontrollers easy to use

What Arduino Adds:

  • USB connection for programming
  • Voltage regulator (5V power)
  • Easy pin headers
  • Reset button
  • Built-in LED (pin 13)
  • Beginner-friendly software

Your RP2040 Robot: Uses the RP2040 microcontroller (dual-core, 133 MHz, much more powerful than Arduino Uno!)

Why Do We Need Motor Controllers?

Problem: Microcontrollers are WEAK!

An Arduino pin can only provide about 40 mA (0.04 amps) at 5V

A DC motor needs 1-5 amps at 12V to spin!

Arduino Pin DC Motor
Voltage 5V 6-12V
Current 40 mA (0.04 A) 1-5 A
Power 0.2 Watts 10-60 Watts

Solution: Motor Driver Chip

Acts as a "power amplifier" - small signal from Arduino controls large power to motor

Note: We call this a "motor driver" - it's the chip that switches power. Complete products like SPARK MAX or Talon contain a motor driver plus extra features like current sensing and controllers.

How Motor Controllers Work

3 Control Signals

  • INA - Direction bit A
  • INB - Direction bit B
  • PWM - Speed control (0-100%)

VNH5019 Features

  • Handles up to 12A continuous
  • Built-in current sensing
  • Thermal shutdown protection
  • Forward, reverse, brake modes

Motor Direction Control

Two digital pins (INA and INB) control motor direction:

INA INB Motor Action Description
LOW HIGH FORWARD Motor spins forward
HIGH LOW REVERSE Motor spins backward
LOW LOW BRAKE Motor stops actively
HIGH HIGH BRAKE Motor stops actively

Example Code Preview (Tomorrow!)

// Drive motor forward
digitalWrite(INA, LOW);
digitalWrite(INB, HIGH);
analogWrite(PWM, 150); // ~60% speed

Hands-On: Arduino IDE

Let's get familiar with the tool you'll use to program your robot!

Step 1: Open Arduino IDE

  • Launch the Arduino application on your computer
  • You should see a blank sketch with setup() and loop()

Step 2: Select Your Board

  • Go to Tools → Board → Arduino Mbed OS RP2040 Boards
  • Select "Arduino Nano RP2040 Connect"
  • This tells the IDE what microcontroller you're using

Step 3: Connect Your Robot (if available)

  • Connect your robot via USB cable
  • Go to Tools → Port and select the port showing your Arduino
  • Don't worry if your robot isn't available yet - we'll just explore the IDE

Your First Code: Blink

This code makes the onboard LED blink on and off. Don't worry about understanding every detail yet - we'll learn it in Lesson 2! 

void setup() {
  // Initialize digital pin LED_BUILTIN as an output
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);   // Turn the LED on
  delay(1000);                        // Wait for 1 second
  digitalWrite(LED_BUILTIN, LOW);    // Turn the LED off
  delay(1000);                        // Wait for 1 second
}

setup() - Runs Once

This section runs one time when the Arduino starts. We use it to configure pins.

loop() - Runs Forever

This section repeats over and over. Our robot's actions go here!

What are functions? setup() and loop() are called functions - reusable blocks of code that perform specific tasks. Every Arduino program needs these two special functions. You'll learn to write your own functions soon!

Copy-Paste Workflow

You'll use this workflow throughout the course to load code examples!

Step 1: Copy the Code

  • Go back to the previous slide with the Blink code
  • Click the green 📋 Copy Code button in the top-right corner
  • The code is now on your clipboard!

Step 2: Open Arduino IDE

  • Click in the Arduino IDE window to make it active
  • You'll see the blank sketch skeleton with setup() and loop()

Step 3: Select All & Paste

  • Press Ctrl+A to select all the existing code in the IDE
  • Press Ctrl+V to paste your copied code
  • The old skeleton code is replaced with the new Blink code!

Step 4: Verify (Check) the Code

  • Click the checkmark ✓ button (Verify) in the top-left of Arduino IDE
  • Arduino checks your code for errors (compiles it)
  • If it says "Done compiling" at the bottom - success!
  • If there are errors, make sure you copied all the code correctly

Practice this now! Go back one slide, click Copy Code, then paste into Arduino IDE.
