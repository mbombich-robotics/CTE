Arduino Code Structure (Review)

Yesterday we saw the structure of Arduino code. Today we'll write our own!

void setup() { // Runs ONCE when Arduino starts // Initialize pins, serial, settings } void loop() { // Runs FOREVER (continuously) // Main program logic goes here }
  • setup() runs once at power-on or reset
  • loop() runs continuously, over and over forever

Copy-Paste Workflow

You'll use this workflow throughout the course to load code examples!

Step 1: Copy the Code

  • When you see a code example on a slide, look for the 📋 Copy Code button
  • Click it to copy the code to your clipboard
  • The code is now ready to paste!

Step 2: Open Arduino IDE

  • Click in the Arduino IDE window to make it active
  • You'll see the blank sketch skeleton with setup() and loop()

Step 3: Select All & Paste

  • Press Ctrl+A to select all the existing code in the IDE
  • Press Ctrl+V to paste your copied code
  • The old skeleton code is replaced with the new code!

We'll practice this workflow next!

Verify and Upload

After pasting code, always verify before uploading!

Step 4: Verify (Check) the Code

  • Click the checkmark ✓ button (Verify) in the top-left of Arduino IDE
  • Arduino checks your code for errors (compiles it)
  • If it says "Done compiling" at the bottom - success!
  • If there are errors, double-check you copied all the code correctly

Step 5: Upload to Arduino

  • Make sure your Arduino Uno is connected via USB
  • Make sure you selected the correct board and port (from yesterday)
  • Click the right arrow → button (Upload) in the top-left
  • Wait for "Done uploading" message

Pro Tip: Always verify before uploading! It catches syntax errors and saves time.

Common Upload Issues

Upload Error?

  • Check that you selected the correct port (Tools → Port)
  • Try unplugging and replugging the USB cable
  • Make sure board type is "Arduino Uno"
  • Some USB cables are power-only - try a different cable

Compile Error?

  • Read the error message - it tells you the line number!
  • Check for missing semicolons (;)
  • Check for matching curly braces { }
  • Make sure you copied all the code

Don't panic! Errors are part of learning. Read the messages and fix one issue at a time.

Your First Program: Blink

Let's make the built-in LED blink!

void setup() { pinMode(13, OUTPUT); // Set pin 13 as output } void loop() { digitalWrite(13, HIGH); // Turn LED on delay(1000); // Wait 1 second digitalWrite(13, LOW); // Turn LED off delay(1000); // Wait 1 second }

Find it: File → Examples → 01.Basics → Blink

Essential Arduino Functions

pinMode(pin, mode)

Configures a pin as INPUT or OUTPUT

pinMode(13, OUTPUT);

digitalWrite(pin, value)

Sets a pin HIGH (on) or LOW (off)

digitalWrite(13, HIGH);

delay(milliseconds)

Pauses the program for specified time (1000ms = 1 second)

delay(1000); // Wait 1 second

Arduino Syntax Rules

  • Semicolons (;) end every statement
  • Curly braces { } group code blocks
  • Case-sensitive - pinMode ≠ PinMode ≠ pinmode
  • Comments start with // (single line) or /* */ (multi-line)
// This is a single-line comment /* * This is a multi-line comment * Useful for longer explanations */ digitalWrite(13, HIGH); // Don't forget the semicolon!

Uploading Your Code

  1. Write or modify code in the Arduino IDE
  2. Click the Verify button (✓) to check for errors
  3. Fix any errors shown in the console (bottom of window)
  4. Click the Upload button (→) to send code to Arduino
  5. Watch for "Done uploading" message

Pro Tip: Always verify before uploading! It catches syntax errors and saves time.

Quiz Preview - Question 1 of 5

Which function runs continuously in Arduino?

A) setup()
B) loop()
C) main()
D) run()

Explanation: The loop() function runs continuously after setup() completes, allowing your Arduino to repeatedly execute code. This is the heart of most Arduino programs!

Quiz Preview - Question 2 of 5

What must you do before using digitalWrite()?

A) Nothing
B) Set pinMode to OUTPUT
C) Call Serial.begin()
D) Add a delay

Explanation: You must configure a pin as OUTPUT using pinMode() before you can write to it with digitalWrite(). This tells the Arduino the pin will be sending signals out, not receiving them.

Quiz Preview - Question 3 of 5

What does delay(1000) do?

A) Wait 1 second
B) Wait 1000 seconds
C) Wait 1 minute
D) Blink LED

Explanation: delay() takes milliseconds as a parameter. Since 1000 milliseconds = 1 second, delay(1000) pauses the program for 1 second.

Quiz Preview - Question 4 of 5

What are the two possible values for digitalWrite()?

A) 0 and 1
B) ON and OFF
C) HIGH and LOW
D) True and False

Explanation: digitalWrite() accepts HIGH (which outputs 5V) or LOW (which outputs 0V) as parameters. While these correspond to 1 and 0 internally, you should always use HIGH and LOW in your code for clarity.

Quiz Preview - Question 5 of 5

What happens if you forget a semicolon at the end of a statement?

A) Nothing
B) Compile error
C) Warning only
D) Arduino crashes

Explanation: Missing semicolons cause the Arduino IDE to fail compilation with an error message. The compiler needs semicolons to know where each statement ends!
