Why Do We Need Variables?

  • Store information - sensor readings, counts, states
  • Reuse values - change one number instead of hundreds
  • Make code readable - "motorSpeed" clearer than "127"
  • Perform calculations - add, subtract, compare

❌ Without Variables

delay(500); delay(500); delay(500); // Change all 3!

✓ With Variables

int delayTime = 500; delay(delayTime); delay(delayTime); // Change once!

Declaring Variables

Every variable needs a type, name, and optional initial value:

int ledPin = 13; // type name value int counter = 0; // Initialize to 0 int buttonState; // No initial value

Naming Rules

  • Start with letter/underscore: motorSpeed, _counter
  • Letters, numbers, underscores: sensor1
  • Case-sensitive: motorSpeed ≠ MotorSpeed
  • No spaces: motor speed ❌
  • No reserved words: int, void ❌

Common Data Types

int

Whole numbers
-32,768 to 32,767

int count = 42;

float

Decimal numbers
Large range

float pi = 3.14159;

boolean

True or False
Only two values

bool isOn = true;

long

Big whole numbers (±2 billion)

long distance = 1000000;

char

Single character (letters, symbols)

char grade = 'A';

Variable Scope: Global vs Local

int globalCounter = 0; // GLOBAL - accessible everywhere void setup() { int setupVar = 10; // LOCAL - only in setup() globalCounter = 5; // Can access global } void loop() { int loopVar = 20; // LOCAL - only in loop() globalCounter++; // Can access global // setupVar NOT accessible here! ❌ }

Global Variables

  • Declared outside functions
  • Accessible everywhere
  • Persist between loop() runs

Local Variables

  • Declared inside functions
  • Only in that function
  • Reset each call

The Serial Monitor

"If you can't see it, you can't fix it!"

  • Debug tool - see what your Arduino is thinking
  • Print values - sensor readings, calculations, states
  • Troubleshoot code - find where things go wrong
  • Monitor progress - track loop iterations

Opening the Serial Monitor

Menu:
Tools → Serial Monitor
Shortcut:
Ctrl+Shift+M (Win) / Cmd+Shift+M (Mac)
Button:
Magnifying glass icon (top right)

Setting Up Serial Communication

Initialize in setup() - only once!

void setup() { Serial.begin(9600); pinMode(13, OUTPUT); } void loop() { Serial.println("Hello!"); delay(1000); }

Baud Rate = Communication Speed

  • 9600 - Standard (most common)
  • 115200 - Faster for lots of data
  • ⚠ Must match Serial Monitor!

Serial.print() vs Serial.println()

Serial.print()

Prints WITHOUT newline

Serial.print("A"); Serial.print("B");

Output: AB

Serial.println()

Prints WITH newline

Serial.println("A"); Serial.println("B");

Output:
A
B

Combining Both

int val = 512; Serial.print("Sensor: "); Serial.println(val);

Output: Sensor: 512

Printing Different Data Types

int counter = 42; float voltage = 5.13; bool isOn = true; void setup() { Serial.begin(9600); } void loop() { Serial.print("Counter: "); Serial.println(counter); // Prints: Counter: 42 Serial.print("Voltage: "); Serial.println(voltage); // Prints: Voltage: 5.13 Serial.print("Status: "); Serial.println(isOn); // Prints: Status: 1 (1=true, 0=false) counter++; // Increment delay(1000); }

Reading Analog Values

Even without a sensor connected, we can read values!

int sensorPin = A0; // Analog pin A0 int sensorValue = 0; void setup() { Serial.begin(9600); } void loop() { sensorValue = analogRead(sensorPin); // Read (0-1023) Serial.print("Analog Value: "); Serial.println(sensorValue); delay(500); // Read twice per second }

Important Notes

  • analogRead() returns 0 to 1023
  • No sensor = "floating" values (electrical noise)
  • Great way to test Serial Monitor setup

Debugging with Serial Monitor

  • Check if code runs - print "Starting..." in setup()
  • Track iterations - print counter each loop
  • Verify conditions - print before/after if statements
  • Monitor sensors - see actual values
  • Find loops - no prints = code stuck
void loop() { Serial.println("Loop start"); // Track execution int value = analogRead(A0); Serial.print("Value: "); Serial.println(value); // See actual value if (value > 500) { Serial.println("High!"); // Know when met } delay(1000); }

Quiz Preview - Question 1 of 5

Which is the correct way to declare an integer variable?

A) var x = 5;
B) int x = 5;
C) integer x = 5;
D) x = 5;

Explanation: Arduino uses 'int' as the keyword for integer variables, followed by the variable name and optional initial value. This declares a variable that can store whole numbers.

Quiz Preview - Question 2 of 5

Which data type stores decimal numbers?

A) int
B) boolean
C) float
D) char

Explanation: The float (floating-point) data type stores decimal numbers like 3.14 or 98.6, while int only stores whole numbers. Use float when you need precision with decimals!

Quiz Preview - Question 3 of 5

What is the most common default baud rate used with Serial.begin()?

A) 300
B) 9600
C) 115200
D) 57600

Explanation: While any baud rate can work if both Serial.begin() and the Serial Monitor match, 9600 is the standard default used in most Arduino examples and tutorials.

Quiz Preview - Question 4 of 5

What's the difference between print() and println()?

A) No difference
B) println adds a newline
C) print is faster
D) println is deprecated

Explanation: Serial.println() automatically adds a newline character after printing, moving to the next line. Serial.print() does not, allowing you to print multiple items on the same line.

Quiz Preview - Question 5 of 5

Where should you declare a variable used in both setup() and loop()?

A) In setup()
B) In loop()
C) Before setup() (globally)
D) Anywhere

Explanation: Global variables (declared before setup()) can be accessed by both setup() and loop(). Variables declared inside a function are only accessible within that function!
