Design Brief
Click to see definition
A document that outlines the problem, constraints, criteria, and requirements for an engineering project. It serves as a roadmap for the design process.
Constraints
Click to see definition
Limitations or restrictions on a design, such as budget, time, materials, size, weight, or other requirements that must be met.
Prototype
Click to see definition
An early working model of a design used to test and evaluate ideas before final production. It helps identify problems and areas for improvement.
Iteration
Click to see definition
The process of repeatedly refining and improving a design based on testing, feedback, and evaluation. Engineering is inherently iterative.
Accuracy vs. Precision
Click to see difference
Accuracy: How close a measurement is to the true value.

Precision: How consistent repeated measurements are with each other.

Safety & Shop Practices

Safety Flashcards

PPE
Click for definition
Personal Protective Equipment - gear worn to protect against hazards, including safety glasses, gloves, hearing protection, and proper clothing.
Emergency Stop Button
Click for details
A large red button (often mushroom-shaped) that immediately cuts power to machinery. Know its location before operating any equipment.
Lockout/Tagout
Click for definition
Safety procedure ensuring machines are properly shut off and cannot start during maintenance or repair work. Prevents accidental startups.
Pinch Point
Click for definition
Any point where two or more objects move together and could trap a body part, causing serious injury. Common on rotating machinery.
Housekeeping
Click for importance
Keeping the work area clean and organized. Good housekeeping prevents trips, falls, and other accidents. Clean as you go!
Safety Data Sheet (SDS)
Click for definition
Document containing information about chemical hazards, safe handling, first aid, and emergency procedures. Required for all chemicals in the shop.

Key Safety Rules

Non-Negotiable Safety Rules

  • Eye Protection: Safety glasses ALWAYS required in the shop
  • Proper Dress: No loose clothing, jewelry, or long hair near machinery
  • One Task: Never operate machinery while distracted or multitasking
  • Ask First: Get instructor approval before using any equipment
  • Report Hazards: Immediately report damaged tools, spills, or unsafe conditions
  • Know Your Limits: Never hesitate to ask for help or clarification

Case Study Lessons

Matthew Henne Incident

  • Young worker injured while operating machinery without proper training
  • Employer failed to discipline employee for safety violations and record incidents
  • Key lesson: Always receive complete training before operating equipment
  • Supervisors must ensure workers are competent and supervised
  • Safety violations must be documented and addressed immediately

Lac-Mégantic Rail Disaster

  • Train derailment caused by inadequate safety procedures and maintenance
  • Key lesson: Following proper procedures prevents catastrophic failures
  • Small shortcuts and neglected maintenance can have devastating consequences
  • Safety culture matters - everyone is responsible

CAD & Manufacturing

CAD Fundamentals Flashcards

CAD
Click for definition
Computer-Aided Design - software used to create precise 2D drawings and 3D models for engineering and manufacturing.
CAM
Click for definition
Computer-Aided Manufacturing - software that converts CAD models into toolpaths and G-code instructions for CNC machines.
G-Code
Click for definition
Programming language for CNC machines. Contains instructions for tool movements, speeds, and positions (e.g., G00 for rapid movement, G01 for linear cutting).
Sketch vs. Extrude
Click for difference
Sketch: 2D drawing on a plane

Extrude: Converting a 2D sketch into a 3D solid by adding depth
Toolpath
Click for definition
The path a cutting tool follows to machine a part. Created in CAM software and includes speeds, feeds, and tool movements.
Rapid Prototyping
Click for definition
Quickly creating physical models from CAD designs using methods like 3D printing, laser cutting, or CNC machining to test and iterate designs.

CAD Drawing Review Checklist

CNC & Manufacturing Key Concepts

CNC Safety Essentials

  • E-Stop Location: Know where emergency stop is before operating machine
  • Review Toolpath First: Always review toolpaths before running on actual machine
  • Secure Workpiece: Material must be firmly clamped - never hold by hand
  • Tool Height Setting: Properly set Z-zero reference point for each tool
  • Clear Work Area: Remove tools, hands, and obstacles before starting
  • Monitor Operation: Stay present and watch the entire machining process

Iterative Design Process

  • Document Everything: Keep iteration logs with photos and measurements
  • Test Fit & Function: Verify parts work as intended before moving on
  • Identify Problems: Note what worked and what didn't
  • Propose Solutions: Explain specific changes for next iteration
  • Implement Improvements: Make calculated design changes
  • Re-test: Verify improvements solved the problems

Arduino Programming & Robotics

Programming Fundamentals Flashcards

Microcontroller
Click for definition
A small computer on a single chip containing CPU, memory, and I/O pins. Arduino is a microcontroller board used for physical computing projects.
setup() vs. loop()
Click for difference
setup(): Runs once when program starts - initialize pins and settings

loop(): Runs repeatedly forever - main program logic
Digital vs. Analog
Click for difference
Digital: Only two states - HIGH (5V) or LOW (0V). Used for buttons, LEDs.

Analog: Range of values (0-1023 on Arduino). Used for sensors, PWM.
PWM
Click for definition
Pulse Width Modulation - rapidly switching a digital pin on/off to simulate analog output. Controls motor speed and LED brightness (0-255).
Serial Monitor
Click for purpose
Tool for debugging and viewing data from Arduino. Use Serial.begin(9600) in setup() and Serial.println() to display values.
Variables
Click for definition
Named storage locations for data. Common types: int (whole numbers), float (decimals), boolean (true/false), String (text).
IR Line Sensor
Click for details
Infrared sensor that detects dark vs. light surfaces by measuring reflected light. Returns HIGH (white) or LOW (black). Used for line following.

Essential Arduino Functions

Core Functions You Must Know

  • pinMode(pin, mode): Set pin as INPUT or OUTPUT
  • digitalWrite(pin, value): Set digital pin HIGH or LOW
  • digitalRead(pin): Read digital pin state (returns HIGH or LOW)
  • analogWrite(pin, value): PWM output on pin (0-255)
  • analogRead(pin): Read analog pin (returns 0-1023)
  • delay(ms): Pause program for milliseconds
  • map(value, fromLow, fromHigh, toLow, toHigh): Scale values between ranges
  • Serial.begin(9600): Initialize serial communication
  • Serial.println(data): Print data to Serial Monitor

Code Challenges

Challenge 1: Spot the Errors

This code has 5 errors. Can you identify them?

1 void setup() { 2 Serial.begin(9600) 3 pinMode(13 OUTPUT); 4 PinMode(9, "OUTPUT"); 5 } 6 7 void loop() { 8 int sensorValue = analogRead(A0); 9 int speed = map(sensorValue, 0, 1023, 0, 300); 10 analogWrite(9, speed); 11 delay(100) 12 }
Click to Reveal Errors
  • Line 2: Missing semicolon after Serial.begin(9600)
  • Line 3: Missing comma between 13 and OUTPUT
  • Line 4: pinMode should be lowercase (case-sensitive!)
  • Line 4: OUTPUT should not be in quotes (it's a constant, not a string)
  • Line 10: PWM values must be 0-255, not 0-300 (will cause overflow)
  • Line 12: Missing semicolon after delay(100)

Challenge 2: What Does This Do?

Read the code and predict what will be printed to the Serial Monitor:

1 int buttonPin = 7; 2 int ledPin = 13; 3 4 void setup() { 5 Serial.begin(9600); 6 pinMode(buttonPin, INPUT); 7 pinMode(ledPin, OUTPUT); 8 } 9 10 void loop() { 11 int buttonState = digitalRead(buttonPin); 12 13 if (buttonState == HIGH) { 14 digitalWrite(ledPin, HIGH); 15 Serial.println("Button Pressed!"); 16 } 17 else { 18 digitalWrite(ledPin, LOW); 19 Serial.println("Button Not Pressed"); 20 } 21 22 delay(500); 23 }
Click to Reveal Answer

Behavior: When the button (on pin 7) is pressed, the LED turns ON and "Button Pressed!" prints to Serial Monitor. When the button is not pressed, the LED turns OFF and "Button Not Pressed" prints. This repeats every 500 milliseconds (half second).

Key Concepts: digitalRead() to check button state, if/else conditional logic, digitalWrite() to control LED, Serial.println() for debugging, and delay() for timing.
