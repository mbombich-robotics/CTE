🔄 ITERATIVE DESIGN PROCESS

Test, Measure, Improve, Repeat

Engineering is Iteration

Professional engineers rarely get designs perfect on the first try. Instead, they embrace iteration—a systematic process of building, testing, learning from failures, and improving until the part works perfectly. This is how SpaceX builds rockets, how Tesla designs cars, and how you'll build championship robots.

Learning Objectives

  • Understand the iterative design methodology
  • Document design decisions and test results
  • Measure dimensional accuracy and functional errors
  • Calculate design adjustments based on measurements
  • Complete Robot Part Iteration Logs (20 points each)
  • Apply lessons learned to improve future designs

Robot Base Project: You'll complete 8 custom parts for your line-following robot. Each part requires an iteration log documenting your design process—worth 160 points total!

📋 ROBOT PART ITERATION LOG

What is an Iteration Log?

An Iteration Log is a structured document that tracks every version of a part from initial concept to final working prototype. It's your engineering notebook—the proof that you followed the design process professionally.

Log Sections

SECTION 1: DESIGN INTENT

What is this part supposed to do?

Dimensions, materials, features

Sketches and CAD screenshots

SECTION 2: MANUFACTURING

How was it made?

CNC toolpaths or 3D print settings

Material used, machine time

SECTION 3: TESTING

Does it fit? Does it work?

Assembly photos

Functional test results

SECTION 4: ANALYSIS

What went wrong?

Measurements of errors

Calculations for corrections

Grading: Each log is worth 20 points. You need 8 logs (one per custom robot part) = 160 points toward your Robot Base Project grade.

💡 SECTION 1: DESIGN INTENT

Clearly State the Purpose

Before you design anything, answer: "What problem does this part solve?"

Example: Motor Mount Bracket

Part Name: Motor Mount Bracket (Left Side)

Purpose: Securely attach DC motor to robot base deck at 90° angle. Must align motor shaft with wheel hub.

Key Dimensions:

  • Motor mounting holes: 31mm × 31mm bolt pattern (M3 screws)
  • Deck mounting: 40mm × 40mm base plate (4× M3 screws)
  • Motor shaft centerline: 15mm above deck surface
  • Material: 3mm PETG (3D printed)

Required Documentation

  • CAD screenshot showing critical dimensions
  • Sketch or reference drawing of mating parts
  • List of tolerances (e.g., "Mounting holes ±0.5mm acceptable")

🏭 SECTION 2: MANUFACTURING DETAILS

Document How You Made It

Future-you (or a teammate) should be able to recreate this exact part from your manufacturing notes.

CNC Parts - Required Info

  • Material: Type, thickness, source (e.g., "3mm polycarbonate, McMaster 8574K12")
  • Machine: OMIO X8-2200 or Crossfire Pro
  • Tool: 6.35mm flat end mill, 2-flute carbide
  • Toolpaths: List operations (2D Pocket for holes, 2D Contour for outline)
  • Feeds/Speeds: 18,000 RPM @ 1500 mm/min, 1.5mm DOC
  • Run Time: Approx. 12 minutes

3D Printed Parts - Required Info

  • Material: PETG, PLA, or ABS (specify brand/color)
  • Nozzle Temp: 240°C (PETG) or 210°C (PLA)
  • Bed Temp: 80°C (PETG) or 60°C (PLA)
  • Layer Height: 0.2mm or 0.3mm
  • Infill: 50% gyroid (structural parts) or 20% grid (lightweight)
  • Supports: Yes/No, tree supports if needed
  • Print Time: 1h 45m

🧪 SECTION 3: TESTING & ASSEMBLY

Fit Test: Does it Assemble?

The most important test: Can you physically assemble this part with the rest of your robot?

Fit Test Checklist

  • 1 Holes align: Do bolt holes line up with mating parts?
  • 2 Clearances: Is there enough space for tools (screwdriver access)?
  • 3 Interference: Does anything hit that shouldn't?
  • 4 Fasteners: Do screws thread smoothly? Nuts fit?

Function Test: Does it Work?

  • Motor mount: Does motor spin freely? No binding?
  • Sensor bracket: Does sensor detect line at correct distance?
  • Battery holder: Does battery fit snugly? Easy to remove?
  • Wheel hub: Does wheel spin without wobble?

Photo Evidence: Include at least 2 photos showing the part installed on your robot. Use a ruler in the photo to show scale.

📏 SECTION 4: MEASUREMENT & ANALYSIS

Quantify What Went Wrong

"The hole is too small" is not useful. "The hole measured 7.8mm instead of 8.0mm (0.2mm undersized)" tells you exactly how to fix it.

Critical Measurements

DIMENSIONAL ERRORS

Measure with calipers:

  • Hole diameters
  • Wall thicknesses
  • Overall dimensions
  • Bolt pattern spacing

FUNCTIONAL ERRORS

Measure performance:

  • Gap sizes (clearance issues)
  • Alignment angles
  • Height above deck
  • Distance from target

Error Calculation Example

Error = Measured Value - Design Value

Error = 7.8mm - 8.0mm = -0.2mm (undersized)

Percent Error = (Error / Design Value) × 100%

% Error = (-0.2mm / 8.0mm) × 100% = -2.5%

🔧 CALCULATING DESIGN CORRECTIONS

Adjust CAD Model for Next Iteration

Once you know the error, update your CAD model to compensate. Add the inverse of the error to get the correct dimension.

Correction Examples

HOLE TOO SMALL

Design: 8.0mm hole

Measured: 7.8mm

Error: -0.2mm (undersized)

Fix: Increase CAD hole to 8.2mm

Logic: 8.2mm design → 8.0mm actual

PART TOO LONG

Design: 100.0mm length

Measured: 100.3mm

Error: +0.3mm (oversized)

Fix: Reduce CAD length to 99.7mm

Logic: 99.7mm design → 100.0mm actual

⚠️ Over-Correction Danger: Don't compensate too much! If your hole was 0.2mm small, add 0.2mm—not 0.5mm "to be safe." That causes new problems.

Iteration 2 Notes

Document your changes:

  • "Increased motor mounting holes from 3.0mm → 3.2mm (0.2mm compensation)"
  • "Reduced base plate length from 100mm → 99.7mm (0.3mm shrinkage observed)"

⚠️ COMMON ERRORS & SOLUTIONS

CNC Router Issues

HOLES TOO SMALL

Cause: Tool deflection, bit runout

Solution: Add 0.1-0.2mm to hole diameters

ROUGH EDGES

Cause: Feed rate too fast, dull tool

Solution: Reduce feed 25%, add finishing pass

TABS TOO THICK

Cause: Tab height too large

Solution: Reduce tab height from 2mm → 1mm

3D Printing Issues

WARPING/LIFTING

Cause: Bed adhesion failure

Solution: Increase bed temp +5°C, use brim

LAYER SEPARATION

Cause: Nozzle temp too low

Solution: Increase nozzle +10°C

HOLES UNDERSIZED

Cause: Plastic expands while printing

Solution: Increase hole diameter +0.3mm

✅ LESSON SUMMARY

Key Takeaways

  • Iteration is the core of engineering—embrace failure as learning
  • Document everything: designs, manufacturing, tests, measurements
  • Measure errors precisely, then calculate exact corrections
  • Each iteration makes your part better—Version 1 is supposed to fail!

The Professional Mindset

"Good engineering is 10% inspiration and 90% iteration. Your first design teaches you what's wrong. Your second design gets closer. By the third, you understand the system well enough to nail it." — Every successful engineer
